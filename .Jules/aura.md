## 2025-04-05 - Missing Dark Mode Configuration in Global CSS
**Learning:** Discovered that while the application actively enforces `<html lang="en" className="dark">` in `src/app/layout.tsx`, the `src/app/globals.css` file completely lacks a `.dark` section to redefine the base HSL CSS variables for dark mode. This causes the application to render dark mode structures using light mode variables, resulting in severe accessibility and aesthetic failures. Additionally, many legacy `.module.css` files are still relying on non-semantic hex variables like `--bg-primary` instead of the modern HSL token system.
**Action:** I will restructure `globals.css` to include rigorous `:root` (light mode) and `.dark` (dark mode) semantic HSL tokens, and systematically refactor the legacy CSS module files to adopt these new semantic tokens globally.
